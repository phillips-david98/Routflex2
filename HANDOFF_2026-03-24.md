## Estado atual do problema (criacao de cliente CRM)
- Sintaxe dos arquivos alterados: OK.
- Mensagem na UI ainda pode aparecer generica ("Erro ao salvar cliente") quando backend retorna erro.
- Pontos mais provaveis para a falha observada:
  1. Backend em modo diferente do esperado (USE_MOCK false sem migration/seeds aplicadas).
  2. Inconsistencia entre URL/porta efetiva da API e frontend em execucao.
  3. Erro de validacao de payload no backend (agora retorna mensagem clara; precisa capturar no Network/log).
  4. Erro de banco no insert (constraint ou schema divergente do payload).

## Como retomar amanha (passo a passo curto)
1. Subir backend CRM e confirmar health em http://localhost:3001/health.
2. Subir frontend CRM e confirmar VITE_API_URL=http://localhost:3001.
3. Reproduzir o cadastro pela UI e capturar:
   - Request/Response do POST /api/customers no DevTools.
   - Linha de erro JSON no terminal do backend (request_id, code, context).
4. Repetir o mesmo payload via PowerShell/curl para isolar frontend vs backend.
5. Se USE_MOCK=false, validar migration/seeds e estrutura da tabela crm_customers.
